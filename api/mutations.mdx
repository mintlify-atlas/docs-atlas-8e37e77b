---
title: "Mutations"
description: "Reference for all GraphQL mutations exposed by the RedEye API."
---

Mutations perform write operations: creating, updating, and deleting campaigns, beacons, annotations, links, and more.

<Warning>
  In Red Team mode, mutations require an active authenticated session. Call `createGlobalOperator` first to establish a session.
</Warning>

---

## Annotations

### `addAnnotationToCommandGroup`

Add a new annotation to an existing command group.

**Returns:** `Annotation!`

<ParamField path="campaignId" type="String" required>
  The campaign the command group belongs to.
</ParamField>

<ParamField path="commandGroupId" type="String" required>
  The command group to annotate.
</ParamField>

<ParamField path="text" type="String" required>
  The annotation body text.
</ParamField>

<ParamField path="user" type="String" required>
  The username of the operator creating the annotation.
</ParamField>

<ParamField path="tags" type="[String!]" required>
  List of tag strings to attach. Pass an empty array if none.
</ParamField>

<ParamField path="favorite" type="Boolean">
  Mark the annotation as a favorite. Optional.
</ParamField>

```graphql
mutation AddAnnotationToCommandGroup {
  addAnnotationToCommandGroup(
    campaignId: "campaign-abc"
    commandGroupId: "cg-001"
    text: "Observed lateral movement via SMB beacon pivot"
    user: "analyst1"
    tags: ["LateralMovement", "SMB"]
    favorite: true
  ) {
    id
    text
    user
    date
    favorite
    tags {
      id
      text
    }
  }
}
```

### `addCommandGroupAnnotation`

Create a new command group from a list of command IDs and add an annotation to it.

**Returns:** `Annotation!`

<ParamField path="campaignId" type="String" required>
  The campaign the commands belong to.
</ParamField>

<ParamField path="commandIds" type="[String!]" required>
  The command IDs to group together.
</ParamField>

<ParamField path="text" type="String" required>
  The annotation body text.
</ParamField>

<ParamField path="user" type="String" required>
  The username of the operator creating the annotation.
</ParamField>

<ParamField path="tags" type="[String!]" required>
  List of tag strings. Pass an empty array if none.
</ParamField>

<ParamField path="favorite" type="Boolean">
  Mark the annotation as a favorite. Optional.
</ParamField>

```graphql
mutation AddCommandGroupAnnotation {
  addCommandGroupAnnotation(
    campaignId: "campaign-abc"
    commandIds: ["cmd-001", "cmd-002", "cmd-003"]
    text: "Credential dumping sequence"
    user: "analyst1"
    tags: ["CredentialAccess"]
    favorite: false
  ) {
    id
    text
    commandIds
    commandGroupId
    commandCount
    generation
  }
}
```

```graphql
    id
    text
    user
    date
    favorite
    tags {
      id
      text
    }
  }
```

## Campaigns

### `createCampaign`

Create a new campaign with the specified parser.

**Returns:** `Campaign!`

<ParamField path="name" type="String" required>
  Display name for the new campaign.
</ParamField>

<ParamField path="creatorName" type="String" required>
  Username of the operator creating the campaign.
</ParamField>

<ParamField path="parser" type="String" required>
  The parser ID to use (obtain from `parserInfo` query).
</ParamField>

```graphql
mutation CreateCampaign {
  createCampaign(
    name: "Operation Nightfall"
    creatorName: "operator1"
    parser: "cobalt-strike-parser"
  ) {
    id
    name
    parsingStatus
    creator {
      id
      name
    }
  }
}
```

```graphql
mutation RenameCampaign {
```

### `deleteCampaign`

Permanently delete a campaign and all its data.

**Returns:** `Boolean!`

<Warning>
  This operation is irreversible. All beacons, commands, annotations, and logs associated with the campaign will be deleted.
</Warning>

<ParamField path="campaignId" type="String" required>
  The campaign to delete.
</ParamField>

```graphql
mutation DeleteCampaign {
  deleteCampaign(campaignId: "campaign-abc")
}
```

---

### `anonymizeCampaign`

Export an anonymized version of a campaign. Returns the path to the exported file.

**Returns:** `String!` — path to the exported archive.

<ParamField path="campaignId" type="String" required>
  The campaign to anonymize and export.
</ParamField>

<ParamField path="anonymizeOptions" type="AnonymizationInput" required>
  Anonymization options. See the [AnonymizationInput type](/api/types#anonymizationinput).
</ParamField>

```graphql
mutation AnonymizeCampaign {
  anonymizeCampaign(
    campaignId: "campaign-abc"
    anonymizeOptions: {
      replaceHostnames: true
      replaceUsernames: true
      replaceDomainsAndIps: true
      removePasswordsHashes: true
      removeScreenshots: false
      removeKeystrokes: false
      removeHidden: true
      findReplace: [
        { find: "ACME-CORP", replace: "TARGET-ORG" }
      ]
    }
  )
}
```

## Authentication

### `createGlobalOperator`

Authenticate to the RedEye server and create or retrieve a global operator session. Used in Red Team mode to establish an authenticated session.

**Returns:** `GlobalOperator` — the authenticated operator, or `null` if authentication fails.

<ParamField path="username" type="String" required>
  The operator username.
</ParamField>

<ParamField path="password" type="String" required>
  The server password.
</ParamField>

```graphql
mutation Login {
  createGlobalOperator(
    username: "analyst1"
    password: "correct-horse-battery"
  ) {
    id
    name
  }
}
```

```graphql
      beaconName
    }
  }
}
```

## Visibility

Beacons, hosts, and servers can be hidden from the UI. These mutations toggle the hidden state.

### `toggleBeaconHidden`

Toggle or set the hidden state for one or more beacons.

**Returns:** `[Beacon!]!`

<ParamField path="campaignId" type="String" required>
  The campaign the beacons belong to.
</ParamField>

<ParamField path="beaconId" type="String">
  A single beacon ID. Use this or `beaconIds`.
</ParamField>

<ParamField path="beaconIds" type="[String!]">
  Multiple beacon IDs. Use this or `beaconId`.
</ParamField>

<ParamField path="setHidden" type="Boolean">
  Explicitly set the hidden value. If omitted, the current state is toggled.
</ParamField>

```graphql
mutation HideBeacons {
  toggleBeaconHidden(
    campaignId: "campaign-abc"
    beaconIds: ["beacon-001", "beacon-002"]
    setHidden: true
  ) {
    id
    beaconName
    hidden
  }
}
```

## Metadata

### `updateBeaconMetadata`

Update display metadata for a beacon — name, color, shape, type, or time of death.

**Returns:** `Beacon!`

<ParamField path="campaignId" type="String" required>
  The campaign the beacon belongs to.
</ParamField>

<ParamField path="beaconId" type="String" required>
  The beacon to update.
</ParamField>

<ParamField path="beaconDisplayName" type="String">
  Custom display name for the beacon.
</ParamField>

<ParamField path="beaconType" type="BeaconType">
  Communication type: `dns`, `http`, `https`, or `smb`.
</ParamField>

<ParamField path="beaconTimeOfDeath" type="DateTime">
  The time the beacon stopped responding.
</ParamField>

<ParamField path="color" type="NodeColors">
  Node color for the graph visualization.
</ParamField>

<ParamField path="shape" type="Shapes">
  Node shape for the graph visualization.
</ParamField>

```graphql
mutation UpdateBeaconMetadata {
  updateBeaconMetadata(
    campaignId: "campaign-abc"
    beaconId: "beacon-001"
    beaconDisplayName: "Foothold - DC01"
    beaconType: smb
    color: red
    shape: diamond
  ) {
    id
    beaconName
    displayName
    meta {
      type
      color
      shape
    }
  }
}
```
