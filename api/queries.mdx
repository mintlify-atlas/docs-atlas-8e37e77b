---
title: "Queries"
description: "Reference for all GraphQL queries exposed by the RedEye API."
---

All queries are read-only operations. Required arguments are marked as such; optional arguments default to the values shown.

<Note>
  Every campaign-scoped query requires a `campaignId` argument. Obtain campaign IDs from the top-level `campaigns` query.
</Note>

---

## `annotation`

Fetch a single annotation by ID.

**Returns:** `Annotation`

<ParamField path="annotationId" type="String" required>
  The ID of the annotation to retrieve.
</ParamField>

<ParamField path="campaignId" type="String" required>
  The campaign the annotation belongs to.
</ParamField>

```graphql
query GetAnnotation {
  annotation(annotationId: "ann-001", campaignId: "campaign-abc") {
    id
    text
    user
    date
    favorite
    tags {
      id
      text
    }
    commandIds
    commandGroupId
    commandCount
    tagCount
    generation
  }
}
```

## `commandGroupIds`

Fetch a list of command group IDs, with optional filters.

**Returns:** `[String]`

<ParamField path="campaignId" type="String" required>
  The campaign to query.
</ParamField>

<ParamField path="beaconId" type="String">
  Filter by beacon.
</ParamField>

<ParamField path="commandGroupIds" type="[String!]">
  Filter to a specific set of command group IDs.
</ParamField>

<ParamField path="commandIds" type="[String!]">
  Filter by command IDs.
</ParamField>

<ParamField path="commandType" type="String">
  Filter by command type string (e.g. `"shell"`)
</ParamField>

<ParamField path="hidden" type="Boolean">
  Include hidden entities. Defaults to `false`.
</ParamField>

<ParamField path="hostId" type="String">
  Filter by host.
</ParamField>

<ParamField path="operatorId" type="String">
  Filter by operator.
</ParamField>

<ParamField path="sort" type="SortTypeComments">
  Sort configuration. Defaults to `{ sortBy: time }`.
</ParamField>

```graphql
query GetCommandGroupIds {
  commandGroupIds(
    campaignId: "campaign-abc"
    beaconId: "beacon-001"
    hidden: false
    sort: { sortBy: time, direction: ASC }
  )
}
```

## `timeline`

Fetch a bucketed summary of active beacons and links with commands over time.

**Returns:** `Timeline!`

<ParamField path="campaignId" type="String" required>
  The campaign to retrieve timeline data for.
</ParamField>

<ParamField path="hidden" type="Boolean">
  Include hidden entities. Defaults to `false`.
</ParamField>

<ParamField path="suggestedBuckets" type="Float">
  Hint for how many time buckets to generate. Defaults to `100`.
</ParamField>

<ParamField path="suggestedStartTime" type="DateTime">
  Start time for the timeline window. Defaults to epoch.
</ParamField>

<ParamField path="suggestedEndTime" type="DateTime">
  End time for the timeline window. Defaults to max date.
</ParamField>

```graphql
query GetTimeline {
  timeline(
    campaignId: "campaign-abc"
    hidden: false
    suggestedBuckets: 50
  ) {
    bucketMinutes
    bucketStartTime
    bucketEndTime
    campaignStartTime
    campaignEndTime
    buckets {
      bucketStartTime
      bucketEndTime
      activeBeacons
      createdBeacons
      dyingBeacons
      deadBeacons
      activeLinks
      createdLinks
      dyingLinks
      deadLinks
      beaconCommandCountPair {
        beaconId
        commandCount
      }
    }
  }
}
```

## Additional queries

The following queries are also available for lower-level log access:

| Query | Description |
|-------|-------------|
| `logs(campaignId, beaconId?, hostId?)` | Fetch raw log entries, optionally filtered by beacon or host. |
| `logsByBeaconId(campaignId, beaconId)` | Fetch all log entries for a beacon in time order, reconstructing the full beacon log. |
| `logFilesByBeaconId(campaignId, beaconId)` | Fetch the list of log file names associated with a beacon. |
| `parsingProgress` | Fetch the current parsing progress across all servers. |
