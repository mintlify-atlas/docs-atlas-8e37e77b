## Object types

### `Campaign`

Represents a RedEye campaign — a collection of C2 servers, beacons, and associated logs.

<ResponseField name="id" type="String!">
  Unique campaign identifier.
</ResponseField>

<ResponseField name="name" type="String!">
  Display name of the campaign.
</ResponseField>

<ResponseField name="beaconCount" type="Int!">
  Total number of beacons in the campaign.
</ResponseField>

<ResponseField name="commandCount" type="Int!">
  Total number of commands across all beacons.
</ResponseField>

<ResponseField name="annotationCount" type="Int!">
  Total number of analyst annotations.
</ResponseField>

<ResponseField name="computerCount" type="Int!">
  Total number of computers (hosts) observed.
</ResponseField>

<ResponseField name="serverCount" type="Int!">
  Total number of C2 servers.
</ResponseField>

<ResponseField name="parsingStatus" type="ParsingStatus!">
  Current state of log file parsing. See [`ParsingStatus`](#parsingstatus).
</ResponseField>

<ResponseField name="firstLogTime" type="DateTime">
  Timestamp of the earliest log entry.
</ResponseField>

<ResponseField name="lastLogTime" type="DateTime">
  Timestamp of the most recent log entry.
</ResponseField>

<ResponseField name="migrationError" type="Boolean!">
  Whether a database migration error occurred for this campaign.
</ResponseField>

<ResponseField name="creator" type="GlobalOperator">
  The operator who created the campaign.
</ResponseField>

<ResponseField name="lastOpenedBy" type="GlobalOperator">
  The operator who most recently opened the campaign.
</ResponseField>

<ResponseField name="parsers" type="[CampaignParser!]">
  The parsers configured for this campaign.
</ResponseField>

---

### `Beacon`

Represents a C2 beacon (implant) running on a compromised host.

<ResponseField name="id" type="String!">
  Unique beacon identifier.
</ResponseField>

<ResponseField name="beaconName" type="String!">
  The name Cobalt Strike assigns to the beacon, or the server name. Unique within a server but not necessarily across a campaign.
</ResponseField>

<ResponseField name="displayName" type="String">
  Custom display name set by an analyst.
</ResponseField>

<ResponseField name="serverId" type="String!">
  ID of the C2 server this beacon is registered with.
</ResponseField>

<ResponseField name="hidden" type="Boolean">
  Whether the beacon is hidden in the UI.
</ResponseField>

<ResponseField name="commandsCount" type="Float!">
  Number of commands run through this beacon.
</ResponseField>

<ResponseField name="commentsCount" type="Float!">
  Number of annotations on this beacon's commands.
</ResponseField>

<ResponseField name="logsCount" type="Float!">
  Number of raw log entries for this beacon.
</ResponseField>

<ResponseField name="host" type="Host">
  The host this beacon is running on.
</ResponseField>

<ResponseField name="meta" type="[BeaconMeta!]!">
  Metadata extracted from the beacon's log lines (IP, PID, process, timing, etc.).
</ResponseField>

<ResponseField name="mitreTechniques" type="[MitreTechniques]">
  MITRE ATT&CK techniques observed via this beacon.
</ResponseField>

---

### `BeaconMeta`

Metadata derived from beacon log `METADATA` lines. A beacon may have multiple metadata records if its state changed over time.

<ResponseField name="id" type="String!">
  Unique identifier.
</ResponseField>

<ResponseField name="type" type="BeaconType">
  Communication protocol used. See [`BeaconType`](#beacontype).
</ResponseField>

<ResponseField name="startTime" type="DateTime">
  Time the beacon first appeared.
</ResponseField>

<ResponseField name="endTime" type="DateTime">
  Time the last command was run.
</ResponseField>

<ResponseField name="ip" type="String">
  IP address of the host at the time of the metadata line.
</ResponseField>

<ResponseField name="port" type="Float">
  Port used by the beacon.
</ResponseField>

<ResponseField name="pid" type="Int">
  Process identifier the beacon is running under.
</ResponseField>

<ResponseField name="process" type="String">
  Process name the beacon is running under.
</ResponseField>

<ResponseField name="username" type="String">
  Username the beacon is running under.
</ResponseField>

<ResponseField name="color" type="NodeColors">
  Node color in the graph visualization. See [`NodeColors`](#nodecolors).
</ResponseField>

<ResponseField name="shape" type="Shapes">
  Node shape in the graph visualization. See [`Shapes`](#shapes).
</ResponseField>

<ResponseField name="source" type="LogEntry">
  The raw log line from which this metadata was extracted.
</ResponseField>

## Input types

### `AnonymizationInput`

Options for the `anonymizeCampaign` mutation. All fields are optional and default to `false` / empty.

<ParamField path="replaceHostnames" type="Boolean">
  Replace all hostname strings with anonymized placeholders. Defaults to `false`.
</ParamField>

<ParamField path="replaceUsernames" type="Boolean">
  Replace all username strings. Defaults to `false`.
</ParamField>

<ParamField path="replaceDomainsAndIps" type="Boolean">
  Replace all domain names and IP addresses. Defaults to `false`.
</ParamField>

<ParamField path="removePasswordsHashes" type="Boolean">
  Strip password and hash strings from log content. Defaults to `false`.
</ParamField>

<ParamField path="removeScreenshots" type="Boolean">
  Exclude screenshot images from the export. Defaults to `false`.
</ParamField>

<ParamField path="removeKeystrokes" type="Boolean">
  Exclude keystroke log files from the export. Defaults to `false`.
</ParamField>

<ParamField path="removeHidden" type="Boolean">
  Exclude hidden beacons, hosts, and servers from the export. Defaults to `false`.
</ParamField>

<ParamField path="findReplace" type="[FindReplaceInput!]">
  List of custom find-and-replace substitutions. Defaults to `[]`.
</ParamField>

---

### `FindReplaceInput`

A single find-and-replace rule for use in `AnonymizationInput.findReplace`.

<ParamField path="find" type="String">
  The string to search for. Defaults to `""`.
</ParamField>

<ParamField path="replace" type="String">
  The string to substitute. Defaults to `""`.
</ParamField>

---

### `SortType`

Sort configuration for command queries.

<ParamField path="sortBy" type="SortOption">
  The property to sort on: `name`, `text`, or `time`. Defaults to `name`.
</ParamField>

<ParamField path="direction" type="SortDirection">
  Sort direction: `ASC` or `DESC`. Defaults to `ASC`.
</ParamField>

---

### `SortTypeComments`

Sort configuration for command group queries.

<ParamField path="sortBy" type="SortOptionComments">
  Sort by `fav`, `time`, or `user`. Defaults to `time`.
</ParamField>

<ParamField path="direction" type="SortDirection">
  Sort direction: `ASC` or `DESC`. Defaults to `ASC`.
</ParamField>

## Enums

### `BeaconType`

The communication protocol used by a beacon.

| Value | Description |
|-------|-------------|
| `dns` | DNS-based C2 communication |
| `http` | Unencrypted HTTP C2 |
| `https` | TLS-encrypted HTTPS C2 |
| `smb` | SMB named-pipe pivot beacon |

---

### `BeaconLineType`

The semantic type of a line within a beacon log file.

| Value | Description |
|-------|-------------|
| `CHECKIN` | Beacon check-in event |
| `ERROR` | Error output from a command |
| `INDICATOR` | Indicator of compromise log line |
| `INPUT` | Operator command input |
| `METADATA` | Beacon metadata line (IP, PID, user, etc.) |
| `MODE` | Mode change (e.g. sleep interval) |
| `OUTPUT` | Command output returned by beacon |
| `TASK` | Task queued by operator |

---

### `LogType`

The category of log file, based on filename conventions.

| Value | Description |
|-------|-------------|
| `BEACON` | Standard beacon interaction log |
| `DOWNLOAD` | File download log |
| `EVENT` | Team server event log |
| `KEYSTROKES` | Keystroke capture log |
| `UNKNOWN` | Unrecognized log file |
| `WEBLOG` | Web server access log |

---

### `ParsingStatus`

The current state of log file parsing for a campaign.

| Value | Description |
|-------|-------------|
| `LIVE_PARSING_CS` | Cobalt Strike live-connect parsing in progress |
| `NOT_READY_TO_PARSE` | Campaign has no servers with logs ready |
| `PARSING_COMPLETED` | Parsing finished successfully |
| `PARSING_FAILURE` | Parsing encountered an unrecoverable error |
| `PARSING_IN_PROGRESS` | Parsing is actively running |
| `PARSING_NOT_STARTED` | Servers are present but parsing has not been triggered |
| `PARSING_QUEUED` | Parsing has been requested and is waiting to start |
