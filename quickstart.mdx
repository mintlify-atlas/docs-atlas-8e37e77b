This guide walks you through downloading RedEye, starting the server, and importing your first campaign.

## Before you begin

RedEye runs as a local server and serves a web UI at `http://127.0.0.1:4000`. You do not need to install Node.js or any dependencies — just download the binary for your platform.

<Steps>
  <Step title="Download RedEye">
    Go to the [GitHub Releases page](https://github.com/cisagov/RedEye/releases) and download the latest binary for your operating system.

    The release package contains:
    - `RedEye` — the main application binary
    - `parsers/` — folder containing parser binaries (e.g. `cobalt-strike-parser`, `brute-ratel-parser`, `sliver-parser`)

    Place both the binary and the `parsers/` folder in the same directory.

    <Warning>
      **macOS only**: When running RedEye for the first time, you may see a "not verified" error. Go to **System Preferences → Security & Privacy → General** and click **Open Anyway**. See the [Apple support page](https://support.apple.com/guide/mac-help/open-a-mac-app-from-an-unidentified-developer-mh40616/) for more details.
    </Warning>
  </Step>

  <Step title="Choose a mode and start the server">
    RedEye runs in two modes. Choose the one that fits your use case:

    <Tabs>
      <Tab title="Red Team Mode">
        Red Team mode enables the full feature set: upload C2 logs, explore data, annotate commands, and create presentations. A password is required.

        ```bash
        ./RedEye --redTeam --password <your_password>
        ```

        On Windows:
        ```bash
        RedEye.exe --redTeam --password <your_password>
        ```
      </Tab>
      <Tab title="Blue Team Mode">
        Blue Team mode is a read-only interface for reviewing campaigns exported by a Red Team. No password required — this is the default mode.

        ```bash
        ./RedEye
        ```

        Or simply double-click the `RedEye` executable.
      </Tab>
    </Tabs>

    The server starts and automatically opens your default browser at [http://127.0.0.1:4000](http://127.0.0.1:4000).

    <Note>
      To stop the server, close the terminal window running RedEye.
    </Note>
  </Step>

  <Step title="Create a user (Red Team mode)">
    In Red Team mode, you will be prompted to log in. To create your first user:

    1. Type a username in the **User** text box.
    2. Click **+ New User** from the dropdown.
    3. Enter the password you set when starting the server.

    You will then be taken to the **Campaign Cards** screen.
  </Step>

  <Step title="Import a campaign">
    From the Campaign Cards screen, click **+ Add Campaign** to upload C2 data.

    You can import one of the provided example datasets to get started right away:

    | Dataset | Format | Download |
    |---|---|---|
    | `gtdataset` | `.redeye` file | [gt.redeye](https://github.com/cisagov/RedEye/raw/develop/applications/redeye-e2e/src/fixtures/gt.redeye) |
    | `smalldata` | `.redeye` file | [smalldata.redeye](https://github.com/cisagov/RedEye/raw/develop/applications/redeye-e2e/src/fixtures/smalldata.redeye) |
    | `gtdataset` | Cobalt Strike logs | [Cobalt Strike folder](https://github.com/cisagov/RedEye/tree/develop/applications/redeye-e2e/src/fixtures/gtdataset) |

    To import a `.redeye` file:
    1. Click **+ Add Campaign**
    2. Select the **.redeye File** tab
    3. Upload the downloaded `.redeye` file
    4. Click **Create Campaign**

    RedEye processes the file and the campaign appears in your campaign list.
  </Step>

  <Step title="Explore the campaign">
    Click on your campaign card to enter the **Explorer** view. From here you can:

    - **Graph** — See the visual network of C2 servers, beacons, and hosts
    - **Hosts tab** — Browse compromised hosts and their associated commands
    - **Beacons tab** — List all C2 agents, with MITRE ATT&CK tags
    - **Timeline** — Replay the attack timeline chronologically
    - **Search** — Press `Ctrl+K` (or `Cmd+K` on macOS) to search across all campaign data
  </Step>
</Steps>
