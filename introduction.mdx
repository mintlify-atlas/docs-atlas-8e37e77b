RedEye is an open-source analytic tool developed by [CISA](https://www.cisa.gov/) and [DOE](https://www.energy.gov/)'s [Pacific Northwest National Laboratory](https://www.pnnl.gov/) to assist Red Teams with visualizing and reporting command-and-control (C2) activities.

RedEye parses logs from C2 frameworks such as Cobalt Strike, Brute Ratel, and Sliver, then presents that data in an interactive, graph-based interface. Operators can tag and annotate commands, replay attack timelines, and export curated campaign data for Blue Team review.

<Info>
  RedEye is released under the [BSD-3-Clause license](https://github.com/cisagov/RedEye/blob/develop/LICENSE) and is free to use, modify, and distribute.
</Info>

## Two modes for two teams

RedEye runs from a single binary but operates in two distinct modes covering both stages of a Red Team engagement.

### Red Team mode

Red Team mode enables the full feature set. You must provide a password to start the server in this mode.

- Upload C2 logs from Cobalt Strike, Brute Ratel, Sliver, or a custom parser
- Explore data in the graph and timeline views
- Annotate commands with comments and tags
- Build slideshow presentations from annotated data
- Export the curated campaign as a `.redeye` file for Blue Team handoff

### Blue Team mode

Blue Team mode is a simplified, read-only interface for reviewing campaigns that a Red Team has curated, annotated, and exported. It runs by default — no password required.

- Review imported `.redeye` campaign files
- Navigate the graph and timeline views
- View Red Team annotations and presentations

<Note>
  Both modes run from the same RedEye binary. Pass `--redTeam --password <password>` to enable Red Team mode, or run without arguments for Blue Team mode.
</Note>

| Feature | Red Team mode | Blue Team mode |
|---|---|---|
| Upload C2 logs | ✓ | ✗ |
| Edit and annotate | ✓ | ✗ |
| Create presentations | ✓ | ✓ |
| Export campaigns | ✓ | ✓ |
| Password required | ✓ | ✗ |

## Supported C2 frameworks

RedEye ships with built-in parsers for the most common C2 frameworks:

- **Cobalt Strike** — Full beacon log parsing including metadata, commands, and output
- **Brute Ratel** — Campaign log parsing and visualization
- **Sliver** — Log ingestion and visualization

You can also [build a custom parser](/parsers/creating-a-parser) for any C2 framework.
